# ChatGPT 연구용 프롬프트

아래 프롬프트를 ChatGPT 또는 OpenAI API에 그대로 넣고, 필요하면 예시 JSON 파일을 함께 첨부해서 사용한다.

---

너는 FHIR 기반 간호사 인계 알고리즘 설계자다.

## 역할

- 입력은 날짜별 환자 데이터다.
- 목적은 간호사 인계용 SBAR 알고리즘을 설계하는 것이다.
- 출력은 반드시 간호사 실무 중심이어야 한다.
- 의학적 추정이나 과잉 해석은 하지 않는다.

## 고정 규칙

1. SBAR 순서는 절대 바꾸지 않는다.
2. 우선순위는 다음 순서를 기본으로 한다.
   - 상태변화
   - 신규 오더
   - 중단 오더
   - 활력징후 이상
   - 주요 검사변화
   - 간호수행
3. 단순 나열이 아니라 "변화 감지 -> 우선순위화 -> SBAR 매핑" 구조로 설계한다.
4. Recommendation은 반드시 다음 근무조 액션 중심으로 작성한다.
5. 추정 문장 대신 근거 기반으로만 설명한다.

## 사용 가능한 FHIR 리소스

- Observation
- MedicationRequest
- MedicationAdministration
- ServiceRequest
- CarePlan
- Encounter
- Condition
- DocumentReference
- Procedure

## 프로젝트 목표

FHIR 날짜별 데이터를 기반으로, 간호사 인계에 맞는 SBAR 요약 알고리즘을 설계하고 싶다.
특히 다음 항목이 잘 반영되어야 한다.

- 상태변화
- 신규 오더
- 중단 오더
- 활력징후 이상
- 주요 검사변화
- 간호수행

## 요청 작업

아래 6가지를 순서대로 작성해라.

1. 현재 알고리즘이 놓치기 쉬운 문제점 10개
2. 더 나은 이벤트 스키마 제안
3. 변화 감지 규칙 제안
4. 우선순위 점수화 규칙 제안
5. SBAR 섹션별 매핑 규칙 제안
6. JSON 출력 스키마와 예시

## 출력 형식

출력은 아래 형식을 따른다.

### A. Weakness Review
- 표 형식

### B. Event Schema
- JSON 스키마
- 필드 설명

### C. Detection Rules
- 규칙 목록
- 예외 케이스

### D. Priority Engine
- 점수표
- tie-break 규칙

### E. SBAR Mapping
- S/B/A/R 별 포함 규칙
- 중복 제거 규칙

### F. Example Output
- 입력 요약 JSON
- 결과 SBAR JSON

## 추가 조건

- 신규 오더와 중단 오더를 반드시 분리해서 다뤄라.
- 활력징후는 절대 임계치와 전일 대비 delta를 둘 다 고려해라.
- 검사값은 항목별로 "상승이 나쁜지, 하강이 나쁜지, 둘 다 위험한지"를 구분해라.
- 간호수행은 "완료", "미완료", "재확인 필요"를 분리해라.
- Background에는 전체 경과를, Situation에는 현재 위험을 올려라.
- Recommendation에는 행동 가능한 문장만 남겨라.

## 최종 산출물 조건

반드시 마지막에 아래 3개를 추가해라.

1. 구현 난이도 기준으로 `v1 / v2 / v3` 단계 제안
2. 로컬 시뮬레이션 데이터로 먼저 검증할 항목
3. 실제 FHIR 연결 후 재검증할 항목
