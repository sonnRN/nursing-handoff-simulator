# Glossary

- handoff engine
  - logic that converts patient snapshots into longitudinal summaries and SBAR output
- harness
  - the shared runtime, fixtures, scripts, and docs used to validate changes safely
- longitudinal summary
  - structured summary across a selected patient date range
- care frame
  - current-management view of isolation, lines, tubes, drains, activity, and related context
- carryover items
  - work that should continue across shifts or date boundaries
- watch items
  - high-risk findings or observations that need close monitoring
- golden fixture
  - stable synthetic test input with expected output patterns
- offline fallback
  - local patient data path used when external FHIR access is unavailable
