# FHIR 기반 간호사 인계 AI 프로젝트 브리프

## 1. 프로젝트 목적

이 프로젝트는 날짜별 FHIR 환자 데이터를 기반으로, 간호사 인계에 바로 사용할 수 있는 SBAR 요약을 생성하는 프로그램이다.

핵심 목표는 다음과 같다.

- SBAR 형식을 유지한다.
- 환자 상태의 "변화"를 중심으로 인계 우선순위를 잡는다.
- 신규 오더, 중단 오더, 활력징후 이상, 주요 검사변화, 간호수행을 빠짐없이 반영한다.
- 간호사가 다음 근무조에 꼭 전달해야 하는 내용만 상단에 올린다.

## 2. 입력 데이터 범위

입력은 FHIR 리소스에서 수집한 날짜별 환자 데이터다.

주요 리소스는 다음과 같다.

- `Observation`: 활력징후, 검사값, I/O, 일부 상태 이벤트
- `MedicationRequest`: 약물 오더
- `MedicationAdministration`: 실제 투약/시행 여부
- `ServiceRequest`: 검사/처치 오더
- `CarePlan`: 간호계획 및 간호 방향
- `Encounter`: 입원 맥락, 병실, 담당의, 입원 배경
- `Condition`: 진단, 문제 목록
- `DocumentReference`: 문서/기록 근거
- `Procedure`: 시술/처치 배경

## 3. 출력 형식

출력은 반드시 `S-B-A-R` 순서를 유지한다.

### Situation

- 현재 가장 중요한 상태변화
- 현재 활력징후 이상
- 오늘 새로 생긴 중요한 오더 또는 중단 오더

### Background

- 선택 기간 동안의 임상 경과
- 날짜순 변화 요약
- 현재 판단에 영향을 주는 배경 정보

### Assessment

- 우선순위 엔진이 뽑은 핵심 문제
- 활력, 검사, 오더를 묶은 임상 의미 단위

### Recommendation

- 다음 근무조가 확인/수행해야 할 액션
- 추적이 필요한 오더
- 미완료 또는 재확인 필요한 간호수행

## 4. 우선순위 원칙

기본 우선순위는 다음 순서를 따른다.

1. 상태변화
2. 신규 오더
3. 중단 오더
4. 활력징후 이상
5. 주요 검사변화
6. 간호수행

단, 실제 구현은 "규칙 게이트 + 점수화" 하이브리드 방식이다.

### 규칙 게이트

아래와 같은 항목은 항상 상단 우선 검토 대상이다.

- 급격한 활력 악화
- 산소/호흡 보조 변화
- line/tube/drain 급변
- 중대한 신규/중단 오더
- 위험 검사 이상

### 점수화

- 상태변화: 100
- 신규 오더: 90
- 중단 오더: 85
- 활력징후 이상: 80
- 주요 검사변화: 75
- 간호수행: 60

여기에 심각도, 최신성, 지속성, 근거 수를 가산한다.

## 5. 현재 프로그램 구조

현재 프로그램은 크게 두 층으로 나뉜다.

- 서버/데이터 조립: `src/server/handlers/patientsApi.js`
- 프론트/SBAR 렌더링: `script.js`

현재 구현의 목표 구조는 다음과 같다.

1. FHIR 정규화
2. 날짜 간 변화 감지
3. 우선순위 엔진
4. SBAR 매핑

## 6. 연구 시 반드시 지켜야 할 제약

- SBAR 순서는 절대 바꾸지 않는다.
- 추정성 문장을 최소화한다.
- 근거 없는 의학적 해석은 하지 않는다.
- 간호 인계 관점으로 설계한다.
- 의사 의사결정 지원 시스템으로 과대 확장하지 않는다.
- 출력은 길어도 "인계용 핵심"이 먼저 보이도록 만든다.

## 7. 좋은 결과의 기준

- 신규/중단 오더를 확실히 구분한다.
- 전일 대비 변화가 바로 보인다.
- Recommendation이 실제 액션 중심이다.
- 사소한 정보가 중요한 정보보다 위에 오르지 않는다.
- 같은 정보가 여러 섹션에 중복 남발되지 않는다.

## 8. 현재 드러난 문제

- 변화 감지가 아직 얇아서 SBAR가 빈약해지는 경우가 있다.
- 검사값 상승/하강의 임상 의미를 항목별로 다르게 해석하지 못한다.
- Recommendation 생성이 약하다.
- 상태변화 규칙이 너무 기계적이라 `Room air` 같은 표현을 부자연스럽게 다룰 수 있다.
- 일부 표시 값 정규화가 일관되지 않아 UI 값과 분석 값이 엇갈릴 수 있다.
